Validation is limited.
Quickstart can verify that each entry points to a non-empty .yml or .yaml file and that the YAML parses successfully.
Quickstart also verifies that each overlay file contains a non-empty top-level overlays: mapping.
Quickstart does not fully validate Kometa overlay file schema yet, so a file that passes here may still fail later and cause Kometa to abort during a run.
Use one or more file, folder, url, git, or repo entries. These are written to library-level overlay_files.
Local file and folder entries are copied into Quickstart-managed config storage after validation, so the saved path may normalize to config/<config_name>/overlay_files/....
Supports mixed local files and folders, raw URLs, Community-Configs git paths, and custom_repo-backed repo paths.
