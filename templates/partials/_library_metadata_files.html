Validation is limited.
Quickstart can verify that each entry points to a non-empty .yml or .yaml file and that the YAML parses successfully.
Quickstart also verifies that each metadata file contains a non-empty top-level metadata: mapping.
Quickstart does not fully validate Kometa metadata file schema yet, so a file that passes here may still fail later and cause Kometa to abort during a run.
Use one or more file, folder, url, git, or repo entries. These are written to library-level metadata_files.
Supports mixed local files and folders, raw URLs, Community-Configs git paths, and custom_repo-backed repo paths.
