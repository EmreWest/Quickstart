Why this matters

Quickstart is meant to be run by you, on your own machine.

It stores your workflow data and credentials locally, and only talks to third-party services when Quickstart needs to validate or use those credentials.

Do not use a Quickstart instance hosted by someone else, because you would be trusting that host with your Plex token, API keys, and local workflow data.

ImageMaid Overview

ImageMaid is a companion cleanup script for Plex metadata. It is designed to help reclaim space from old custom posters, backgrounds, and related image files without forcing a full Plex metadata reset.

Quickstart focuses on the most common setup pieces from the ImageMaid wiki: choosing a mode, supplying the Plex path, deciding how to access the Plex database, and enabling optional cleanup actions.

What Quickstart helps you manage

report previews changes, move stages removable files into the restore area, and restore brings staged files back. Quickstart also exposes optional operations like PhotoTranscoder cleanup, Empty Trash, Clean Bundles, and Optimize DB.

What ImageMaid needs

The key requirement is your Plex data path, meaning the directory that contains folders such as Cache, Metadata, and Plug-in Support. For database access, ImageMaid can either connect with your Plex URL and token or copy the local database from that Plex path.

Recommended usage

The wiki recommends not running ImageMaid while tools like Kometa or TitleCardMaker are actively changing posters. If you use the local Plex database, Plex should be idle first so the selected artwork state is fully written to disk.

Read the full ImageMaid wiki page for install details, scheduling, and the complete option list.

Setting up Test Libraries

While going through this process, Kometa is going to load the items in your library, create some collections, and apply some overlays. If you have a large library, this will be very time-consuming.

For learning and testing, it is advisable to create a small test library of a reasonable size—preferably below a thousand items.

You may choose to take advantage of the plex-test-libraries repository, which contains pre-made videos to use when following this guide.

Using the plex-test-libraries repository will ensure enough variety in media to populate the example collections. Running some of these default collections against a library of a few thousand movies can take hours, and for iterative testing, it's useful to have something that will run in a few minutes or seconds.

To clone the repository, navigate to your desired storage location and run:

git clone https://github.com/chazlarson/plex-test-libraries
cd plex-test-libraries

You should now have two folders, test_tv_lib and test_movie_lib. Mount each of these as a library within Plex and use them in Quickstart.

Alternatively, for your convenience (ONLY On Local-* installations of Quickstart) use the button on the Quickstart page to clone the repository to a directory right beside Quickstart. plex-test-libraries is about 7GB in size, so be patient while it downloads.
